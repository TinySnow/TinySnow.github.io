# 把话说清楚，比把话说漂亮更值钱

　　你有没有过这种体验：看了一个演讲，特别精彩，金句频出，但你走出会场后一个字都记不住。

　　反而是某个同事跟你说了三句大白话——"这事大概需要两周，因为卡在审批那块，我们要先搞定接口协议"——你脑子里清清楚楚。

　　一个是用词漂亮的雾，一个是用词朴素的蓝图。前者让你情绪起伏，后者让你行动。

　　我今天想告诉你一个可能违背你直觉的事：在现实世界里，精准比优美有交易价格。

　　你去看任何一份高薪岗位的职责描述，它找的从来不是"会说话"的人，而是"能把复杂事情说清楚"的人。那个"清楚"就是精准。

## 一、模糊语言是成本外部化的最高效工具

　　模糊话术有一个超能力：它把理解成本完全转移给对方。

　　你说"这个项目我们要优化整体体验"。"优化体验"是什么意思？加颜色？换图片结构？重做数据流？还是让页面快一百毫秒？

　　你一句话没说清楚，却把这句话的翻译任务甩给了一屋子人。他们每个人按自己的理解去执行，做出来的东西肯定五花八门。

　　而你在总结会上还能说："我不是这个意思。"

　　这种用模糊语言保护自己的习惯，在知识密集行业里是极高昂的隐性成本。你花了整个团队的时间去翻译一个本来可以清楚说出来的意图。

　　你把模糊感留给自己，把认知负荷全转移出去。这不是表达策略，这是**责任外包**的最隐蔽形式。

## 二、精确不是枯燥，是分发效率上的降维打击

　　程序员都知道，一个 API 的描述越精确，使用者犯错的概率越低，整体联调的效率越高。

　　人类协作中也是一样。一句话说清楚"目标是什么、截止什么时候、验收标准是什么"，整个组的精度会瞬间提高一大截。

　　**说话清楚这件事，本质上是在为别人节省运行内存。**

　　你把指令的每一前提和范围定义清楚，对方就不需要猜测。不需要猜测的协作消耗的心理能量远小于需要猜测的。

　　这个差别在大型组织里会产生指数级的效应——一个清晰的上游指令能让下游几百人少跑无数弯道。而一个模糊的指令会导致下下级层层放大歧义，最后整个执行结果跟你最初的想象风马牛不相及。

## 三、为什么人们害怕说太明白——害怕担责

　　你要是说得很清楚："需要在 7 月 15 日之前完成移动端重构，验收标准是页面不超过六层嵌套、首屏加载低于 1.8 秒"——万一没做成，你就被锁定了。

　　但如果你说"我希望它可以更流畅一点"，你会永远安全。

　　这是模糊表达真正的利益点——它保护你的言说信誉，却暗中破坏了实际推进的质量。

　　冷一点说：**你害怕"说清楚之后会翻车"，所以选择了永远不真正发车。**

　　一个从没有开过车前灯的人比不上一辆在坑坑洼洼但大灯亮着的小皮卡。

## 四、怎么练精准：改书面，改请求，改拒绝

　　改书面表达。下次写邮件时，先看一遍是不是有"尽快"、"大概"、"可能"这种词出现。如果有，每个都改成具体的时间、数量、参考值。

　　改请求。如果你需要别人做一件事，练习在第二句话就指明可核查的交付内容。不是"帮我看看这个方案"，而是"帮我看第三页之后的实现路径那部分"。

　　改拒绝。不找"最近太忙了"的借口，换成"到下个月底之前，我不能接新任务，之后可以再来找我"。一个可以确定的拒绝，远好过一个不可确定的模糊承诺。

　　这三样练四个月，你会发现别人对你的信任度和配合度都在提高，因为他们终于开始觉得跟你合作是可预测的。可预测在当代就是巨大的信任红利。

## 五、精确说话等于不占用别人的 CPU 周期

　　你可能从来没从这个角度想过。每一个跟你对话的人，大脑都在分配认知资源——他在想你到底什么意思，他想猜你是不是不高兴，他在解读你的言外之意。

　　你每扔出去一句模糊话，对方的那台处理器就多消耗一次算力。如果你是管理者，你的团队每天花 15% 的时间在"猜老板到底什么意思"，这 15% 是你买的单，不是他们喜欢的。

　　漂亮会赢得喝彩，清楚会赢得信任。前者是瞬时变量，后者是复利。你说你要哪个？

　　下次开口前，先问自己一句：我接下来要说的话，有没有一个明确的接收标准？对方听完能不能立刻知道接下来该做什么？如果答案是否定的，这句话可能还不到该说的时候。

　　这个习惯练久了你会发现一个意外的好处：你话说得越清楚，越没人敢跟你打太极，嘛。因为太极只有在模糊地带才能打。
